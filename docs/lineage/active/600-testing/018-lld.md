# Implementation Spec: 0600 - AST-Based Import Sentinel

## 1. Objective
Enhance mechanical validation to strictly catch "Lingering Symbols" (missing imports or undefined variables) before execution using AST analysis.

## 2. Proposed Changes

### 2.1 Files to Modify
| File | Change Type | Description |
|------|-------------|-------------|
| `assemblyzero/workflows/requirements/nodes/validate_mechanical.py` | Modify | Integrate the new AST Sentinel check. |
| `assemblyzero/utils/ast_sentinel.py` | Add | Core AST parsing logic and visitor. |
| `tests/unit/test_ast_sentinel.py` | Add | Unit tests for AST Sentinel logic. |

## 3. Requirements
1. Parse using `ast.parse` and `NodeVisitor`.
2. Detect `ast.Load` without corresponding definition.
3. State specific error: "Symbol '{name}' used on line {line} but not imported."
4. Strictly fail gate (`exit 1`) on un-ignored errors.
5. Recursive stack-based scope tracking (nested scopes, comprehensions, walrus).
6. Ban star imports explicitly.
7. Support `if TYPE_CHECKING:` blocks.
8. Support `# sentinel: disable-line`.

## 4. Technical Strategy
1. **ast_sentinel.py:** Implement `SymbolSentinel` using `ast.NodeVisitor`.
   - Maintain a `scope_stack` (list of sets) to track definitions.
   - `visit_Import`/`visit_ImportFrom`: Register aliases in the current scope. Reject `*`.
   - `visit_FunctionDef`/`visit_ClassDef`: Push new scope, register args/name, pop on exit.
   - `visit_Name(ctx=ast.Load)`: Check if name exists in any scope in the stack or `builtins`.
   - Handle `TYPE_CHECKING` blocks by visiting their bodies even if not executed.
   - Handle `# sentinel: disable-line` by checking `tokenize` or line-based string match.
2. **Integration:** Update `validate_mechanical.py` to call `ast_sentinel.analyze_file(path)`.
   - If errors found, print to `sys.stderr` and `sys.exit(1)`.

## 6. Implementation Instructions

### 6.1 assemblyzero/utils/ast_sentinel.py
- Define `SentinelError` namedtuple (symbol, line, message).
- Implement `SymbolSentinel(ast.NodeVisitor)`.
- Use `builtins.__dict__.keys()` for the initial global scope.
- `visit_NamedExpr` (Walrus): Register target in the nearest non-comprehension scope.

### 6.2 tests/unit/test_ast_sentinel.py
- Test happy paths (imports, functions).
- Test failing paths (missing symbol, star import).
- Test scope isolation (comprehension variables not leaking).
- Test bypass mechanism.
