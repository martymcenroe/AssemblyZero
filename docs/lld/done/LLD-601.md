## 1. Context & Goal
* **Issue:** #601
* **Objective:** Eliminate the "PowerShell Trap" where Bash-style commands are intercepted by PowerShell on Windows, leading to token waste and execution failures. Also enhance Claude quota detection.
* **Status:** APPROVED (Fast-tracked via Babysit Protocol)

## 2. Proposed Changes

### 2.1 Files Changed
| File | Change Type | Description |
|------|-------------|-------------|
| `assemblyzero/utils/shell.py` | Add | New utility for cross-platform command execution |
| `assemblyzero/core/errors.py` | Modify | Added Claude CLI usage limit patterns |
| `assemblyzero/core/llm_provider.py` | Modify | Updated ClaudeCLIProvider to respect usage limits |
| `tests/unit/test_shell_util.py` | Add | Unit tests for shell wrapping logic |

### 2.2 Technical Approach
1. **Shell Utility:** Created `wrap_bash_if_needed(command)` which detects Bash symbols (`&&`, `2>&1`, etc.) on Windows and automatically wraps the command in `bash -c '...'`.
2. **Safe Runner:** Created `run_command()` as a drop-in replacement for `subprocess.run()` that handles platform-specific wrapping and shell-escaping.
3. **Claude Quota Detection:** Added "usage limit", "usage has been exhausted", and "wait until" to `_BILLING_PATTERNS`. This ensures that when Marty's Claude Pro plan hits its rolling 5-hour limit, the system identifies it as a fatal error and trips the circuit breaker rather than retrying.

## 3. Requirements
- [x] Automatic `bash -c` wrapping for complex commands on Windows.
- [x] Correct escaping of single quotes in wrapped commands.
- [x] Identification of Claude Max subscription usage limits.
- [x] Unit tests for Windows and Unix behavior.
