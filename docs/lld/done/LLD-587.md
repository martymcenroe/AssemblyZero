## 1. Context & Goal

* **Objective:** Implement a mechanical safety gate to prevent the TDD workflow from silently committing truncated files, and enforce a token-saving Two-Strike retry rule with context pruning.

### 2.2 Technical Approach
1. **Two-Strike Rule:** `MAX_FILE_RETRIES` reduced from 3 to 2.
2. **Context Pruning:** When retrying a file, the `completed_files` list is emptied. The prompt only includes the LLD and the current file, saving ~88% of retry tokens.
3. **Size Gate:** `validate_code_response` accepts `existing_content`. If the file is >10 lines and the new code shrinks by >50%, it returns a validation error, triggering the Two-Strike retry loop automatically.
