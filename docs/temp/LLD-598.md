## 1. Context & Goal
* **Issue:** #598
* **Objective:** Implement a mechanical "firewall" for shell commands to prevent agents from using dangerous or unauthorized flags (like `--admin`, `--force`, `-D`).
* **Status:** APPROVED (Fast-tracked via Babysit Protocol)

## 2. Proposed Changes

### 2.1 Files Changed
| File | Change Type | Description |
|------|-------------|-------------|
| `assemblyzero/utils/shell.py` | Modify | Added `PROHIBITED_FLAGS` and `validate_command` check |
| `tests/unit/test_shell_security.py` | Add | Unit tests for command blocking |

### 2.2 Technical Approach
1. **Deny List:** Hard-coded `PROHIBITED_FLAGS = ["--admin", "--force", "-D", "--hard"]`.
2. **Middleware:** Integrated `validate_command(command)` into `run_command()`. 
3. **Action:** If a prohibited flag is detected in either a string or list-based command, a `ValueError` is raised with a "Security Block" message.
4. **Integration:** All future workflow nodes using `run_command` are automatically protected by this firewall.

## 3. Requirements
- [x] Hard-block dangerous flags.
- [x] Support both string and list command formats.
- [x] Unit tests verify blocking of all prohibited flags.
- [x] Safe commands pass through correctly.
